Herencia de métodos


public class Vehiculo {

  private String patente;
  private String marca;
  private int millasNauticas;

  public Vehiculo() {
    //Constructor
  }

  public void transportarse() {
    millasNauticas = millasNauticas*10;
  }
}

public class Camion extends Vehiculo {
    private int cargaMaxima;
    private int kmsRecorridos;
    
    public Camion(){
        //Constructor
    }
    
    public void transportarse() {
        kmRecorridos = kmRecorridos*5;
    }
    
    public void activarTolva() {
        //Activa tolva
    }
}

  • Camión al ser una extensión de su clase “padre” Vehículo, hereda todos sus atributos, por lo que no vuelven a escribirse en la clase Camión

  • El método “transportarse”, es un método heredado, pero al escribirlo de nuevo en Camión estamos especificando que modificamos el funcionamiento que tiene originalmente

    • Si no se vuelve a escribir, Camión hereda igualmente el método pero el funcionamiento de este se mantiene a como fue especificado en su “padre”

  • Además, Camión al ser una clase más específica que su “padre” Vehículo, ha agregado un nuevo método que le permite activar una tolva
